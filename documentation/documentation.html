CyberSafe Documentation

Everything you need to set up, customize, and launch your CyberSafe website. No prior experience required beyond basic HTML editing.

01 Introduction

CyberSafe is a premium, fully responsive HTML template designed for cybersecurity companies, VPN services, and IT security startups. It features a modern dark aesthetic with neon accents, smooth animations, and a complete set of sections ready to customize.

What's Included

  • index.html — Main template file, fully structured and commented
  • style.css — All styles with CSS custom properties for easy theming
  • script.js — Interactions: navbar scroll, counters, pricing toggle, form handling
  • documentation/ — This file you're reading right now

Template Sections

  • Navigation bar (sticky, responsive, hamburger menu for mobile)
  • Hero section with animated shield and particle grid
  • Trusted brands marquee bar
  • Features grid (6 cards with animated progress bars)
  • How It Works (3 steps + animated terminal)
  • Pricing (3 plans, monthly/annual toggle)
  • Testimonials (3 cards)
  • Contact form (client-side, easily connected to any backend)
  • Footer with links and branding

02 Requirements

CyberSafe is a pure HTML/CSS/JavaScript template. There is no build process, no Node.js, no npm. All you need is:

  • A modern web browser (Chrome, Firefox, Safari, Edge)
  • A text editor to customize content (VS Code recommended)
  • A web host or static hosting service to deploy (GitHub Pages, Netlify, Vercel, any shared hosting)
✓ No frameworks, no dependencies to install. Everything works out of the box by opening index.html in a browser.

External Dependencies (loaded via CDN)

The following libraries are loaded automatically from CDN — no local installation needed:

LibraryVersionPurpose
AOS2.3.1Animate on Scroll — fade/slide effects when sections enter the viewport
Google Fonts—Rajdhani (headings), Share Tech Mono (monospace), Inter (body)
⚠ The template requires an internet connection to load fonts and AOS from CDN. For fully offline use, download these assets and reference them locally.

04 Quick Start

1. Open and preview locally

Simply open docs/index.html in your browser. The template will work immediately including all animations. All CDN assets load automatically.

2. Edit your content

Open docs/index.html in a text editor and look for the section comments to find what to replace:

<!-- ░░ HERO ░░ -->
<!-- ░░ FEATURES ░░ -->
<!-- ░░ HOW IT WORKS ░░ -->
<!-- ░░ PRICING ░░ -->
<!-- ░░ TESTIMONIALS ░░ -->
<!-- ░░ CONTACT ░░ -->
<!-- ░░ FOOTER ░░ -->

3. Deploy

Upload the three files inside docs/ to any web host. For a free live demo, GitHub Pages works perfectly — just push to a repo and enable Pages in the repository settings.

ℹ Make sure index.html, style.css and script.js are all in the same folder when deploying. The HTML references the other files relatively.

05 Colors & Theme

All colors are defined as CSS custom properties at the top of style.css. Change them once and the entire template updates automatically.

/* style.css — line 1 */
:root {
  /* Backgrounds */
  --bg: #080c10;       /* Page background */
  --bg2: #0d1117;      /* Cards, sections */
  --bg3: #111820;      /* Inputs, deep contrast */

  /* Brand / Accent */
  --neon: #00ffaa;     /* Primary accent (green-cyan) */
  --neon2: #00cfff;    /* Secondary accent (blue) */
  --neon-dim: rgba(0,255,170,0.12);   /* Tinted backgrounds */
  --neon-border: rgba(0,255,170,0.25); /* Glowing borders */

  /* Text */
  --text: #d0dce8;      /* Body text */
  --text-muted: #5a7080; /* Secondary text, labels */
  --white: #f0f6ff;     /* Headings, highlights */
}

Example: Change to a blue/purple theme

:root {
  --neon: #7c3aed;     /* Purple */
  --neon2: #2563eb;    /* Blue */
  --neon-dim: rgba(124,58,237,0.12);
  --neon-border: rgba(124,58,237,0.25);
}
✓ When changing --neon, always update --neon-dim and --neon-border to match using the same RGB values with reduced opacity.

Changing fonts

To change a font, replace the Google Fonts URL in <head> and update the CSS variable:

<!-- In index.html <head> -->
<link href="https://fonts.googleapis.com/css2?family=Orbitron:wght@400;700&display=swap" rel="stylesheet">

/* In style.css :root */
--font-display: 'Orbitron', sans-serif;

07 Sections Guide

Hero — Stats Counter

The animated counters in the hero are driven by data-count attributes. Change the end values here:

<span class="stat-num" data-count="99.9">0</span> <!-- 99.9% Uptime -->
<span class="stat-num" data-count="256">0</span>  <!-- 256-bit -->
<span class="stat-num" data-count="140">0</span>  <!-- 140+ Servers -->
<span class="stat-num" data-count="50000">0</span>
✓ Numbers with a decimal point (like 99.9) automatically display with one decimal place. All others display as integers.

Features — Progress Bars

Each feature card has a progress bar controlled by inline style="width:XX%". The JS animates it when the card scrolls into view:

<div class="feat-bar-fill" style="width:96%"></div>
<span class="feat-percent">96% threat coverage</span>

Logos / Trust Bar

The scrolling logos are plain text inside .logos-track. Replace the <span> elements with your client names. The track is duplicated to create a seamless infinite loop — make sure to update both sets:

<div class="logos-track">
  <!-- First set -->
  <span>Accenture</span><span>Deloitte</span>...
  <!-- Duplicate set (for seamless loop) -->
  <span>Accenture</span><span>Deloitte</span>...
</div>

Contact Form

The form currently simulates a submission with a 1.2s delay. To connect it to a real backend (e.g. Formspree, EmailJS, or your own API), replace the handleSubmit function in script.js:

// script.js — replace the setTimeout block with a real fetch:
async function handleSubmit(e) {
  e.preventDefault();
  const data = {
    name: document.getElementById('name').value,
    email: document.getElementById('email').value,
    message: document.getElementById('message').value
  };
  const res = await fetch('https://formspree.io/f/YOUR_ID', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  });
  if (res.ok) document.getElementById('formSuccess').classList.remove('hidden');
}

08 Pricing Toggle

Each plan card's .amount element holds both price values as data attributes. The JavaScript reads these on toggle:

<span class="amount" data-monthly="29" data-annual="20">29</span>

To update prices, just change data-monthly, data-annual, and the initial text content. No JS changes needed.

PlanMonthly defaultAnnual default
--data-monthly="9"data-annual="6"
--data-monthly="29"data-annual="20"
--data-monthly="99"data-annual="69"

09 Animations

Scroll animations use AOS (Animate on Scroll). Any element with a data-aos attribute will animate when it enters the viewport.

Available effects

<!-- Available data-aos values -->
data-aos="fade-up"
data-aos="fade-down"
data-aos="fade-right"
data-aos="fade-left"
data-aos="zoom-in"
data-aos="flip-left"

<!-- Delay in milliseconds -->
data-aos-delay="200"

<!-- Duration override -->
data-aos-duration="1000"

Disable animations

To turn off all scroll animations (e.g. for performance), remove the data-aos attributes or change the AOS init in script.js:

AOS.init({ disable: true }); // disables all AOS

CSS-only animations

The following animations are pure CSS and always run regardless of AOS:

  • Shield rings — @keyframes spin-ring in style.css
  • Scan line — @keyframes scan
  • Logo marquee — @keyframes marquee
  • Badge pulse dot — @keyframes pulse-dot
  • Terminal cursor blink — @keyframes blink
